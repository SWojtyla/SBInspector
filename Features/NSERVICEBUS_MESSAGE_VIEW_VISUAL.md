## Message Intent Color Coding

The Message Intent property uses color-coded chips for easy identification:

| Intent    | Color     | Visual      |
|-----------|-----------|-------------|
| Send      | Blue      | [Send]      |
| Publish   | Green     | [Publish]   |
| Subscribe | Cyan      | [Subscribe] |
| Reply     | Grey      | [Reply]     |

## User Flow

### First Time Using NServiceBus View

1. User opens a message details view
2. System detects NServiceBus properties in the message
3. Toggle switch appears in header (default: OFF)
4. User clicks toggle to ON
5. View immediately switches to NServiceBus layout
6. Preference is saved to storage

### Subsequent Messages

1. User opens another NServiceBus message
2. System loads saved preference (ON)
3. Message automatically displays in NServiceBus view
4. No need to toggle again

### Switching Back to Standard View

1. User toggles switch to OFF
2. View switches to standard layout
3. Preference is updated
4. Future messages open in standard view

### Non-NServiceBus Messages

1. User opens a regular Service Bus message
2. No toggle appears (message doesn't have NServiceBus properties)
3. Standard view is shown automatically
